Built on the same philosophy

Both workspaces share safety boundaries designed for real personal data.

🔒

Local & private

Data stays on your machine in JSON, CSV, Markdown, or SQLite. Nothing is sent to a third-party service.

👁️

Read-only by design

Agents observe and structure information. They do not transfer money, send email, or change account settings.

🤖

Agent-native

Prompts, rules, and MCP servers are first-class. Run in Cursor, VS Code Copilot Agent Mode, or Claude.

📊

Structured output

Schemas, validation, and confidence gates turn messy sources into data you can query, export, and browse.

🏦 bank-atlas

Local, no-app personal finance workflow for an AI coding agent

A repeatable MCP workspace where an agent reads your own bank pages through Playwright MCP (with you in the loop for login and 2FA), normalizes the data with zero-dependency Node tooling, and produces a local financial snapshot — Markdown, JSON, CSV, and an optional read-only web dashboard.

  • Visible browser extraction with manual auth handoff
  • Statement CSV ingest and schema validation
  • Local dashboard at 127.0.0.1:4173
  • Optional SQLite history and spending trends

📨 mail-atlas

Local, privacy-first AI workspace for promotional inbox intelligence

Turns promotional email into structured, searchable offer data without modifying, sending, or deleting mail. A read-only mail MCP server retrieves messages locally; agent prompts classify and extract offers (including image-aware parsing), validate confidence, and store results in SQLite — with an optional read-only web UI.

  • Read-only mail MCP (IMAP, Gmail, Microsoft Graph)
  • Multi-step agent pipeline with confidence gating
  • Calendar and list views in offers-web
  • Image analysis for banner and attachment offers

How it works

The same pattern: agent + MCP tools → structured local files → optional browser UI.

1

Connect tools

Register MCP servers in Cursor or VS Code — Playwright for banks, mail-mcp-server for inbox.

2

Run agents

Follow prompts and rules. You stay in control for login, 2FA, and anything sensitive.

3

Build locally

Normalize into schemas, validate, audit for secrets, and export Markdown, JSON, or CSV.

4

Browse offline

Open the local dashboard to review snapshots or offers — read-only, bound to localhost.
